# App móvil DM

## Arquitectura del proyecto
El sistema está organizado de forma modular para separar responsabilidades y facilitar el mantenimiento del proyecto.

## Repositorios del proyecto
- [Frontend móvil](https://github.com/dayana2100/App-dm-frontend)
- [Backend](https://github.com/dayana2100/App-dm-backend)
- [Base de datos](https://github.com/dayana2100/App-dm-database)

## Tecnologías principales
- Flutter
- Dart
- Node.js
- Express
- Prisma
- MySQL

## Organización del proyecto
- **Frontend:** aplicación móvil desarrollada en Flutter.
- **Backend:** API y lógica de negocio.
- **Base de datos:** scripts, modelo de datos y documentación técnica.

## Convención de ramas
- `feature/HU-001-login`
- `feature/HU-002-registro`
- `feature/HU-003-home`

## Convención de commits
- `feat:` nueva funcionalidad
- `fix:` corrección de errores
- `docs:` documentación
- `refactor:` mejoras internas
- `test:` pruebas
- `chore:` mantenimiento
